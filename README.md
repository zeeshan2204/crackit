## 📌 What is CrackIt?

CrackIt simulates real campus placement exams (TCS, Infosys, Wipro style) with 4 sections — **Numerical Ability, Verbal Ability, Reasoning, and Coding**. After the exam, it generates an **AI-powered performance analysis** with a personalized 7-day study plan using Groq's LLaMA3.

Built as a final year project with a production-grade tech stack — deployed live on Vercel + Railway + Neon.

---

## ✨ Features

### 🎯 Exam Engine
- **4-section placement exam** — Numerical · Verbal · Reasoning · Coding
- **120+ MCQ questions** across difficulty levels (Easy / Medium / Hard)
- **12 coding problems** with live test case execution (Python, Java, C, C++)
- **5 random questions** picked per MCQ section every attempt — never the same exam twice
- **1-hour global countdown timer** — runs continuously across all sections
- **End Exam Early** — save partial progress and see results anytime

### 🤖 AI Performance Analysis
- Personalized feedback generated by **Groq LLaMA3** after exam completion
- Covers: Overall assessment · Strengths · Focus areas · 7-day study plan
- Section-wise and difficulty-wise (Easy/Medium/Hard) performance breakdown
- Weak area action plans with specific TCS/Infosys preparation tips

### 📋 Answer Review
- Post-exam review of all MCQ sections
- ✅ Correct answers highlighted green
- ❌ Wrong answers highlighted red with correct answer shown
- ○ Skipped questions with correct answer revealed
- Difficulty tag and topic shown per question

### 💻 Live Code Execution
- Write and run code directly in the browser
- Executed on local backend via **Python subprocess** — no external API needed
- Real test case validation with PASS/FAIL output
- Supports **Python · Java · C · C++**

### 📊 Progress & Leaderboard
- Score trend chart (Recharts line graph) across multiple attempts
- Stats: Best score · Latest score · Average · Trend
- Section-wise score history table
- Global leaderboard ranking all users by total score

### 🔐 Authentication
- JWT-based auth with Access + Refresh token flow
- Email/Password registration and login
- **Demo Mode** — try the full exam without signing up (no account needed)

## 🚀 Setup & Installation

### Prerequisites
- Python 3.11+
- Node.js 18+
- PostgreSQL (or [Neon](https://neon.tech) free tier)
- Groq API key — free at [console.groq.com](https://console.groq.com)

### 1. Clone the repo
```bash
git clone https://github.com/zeeshan2204/crackit.git
cd crackit
```

### 2. Backend setup
```bash
cd backend

# Create virtual environment
python -m venv venv
venv\Scripts\activate          # Windows
source venv/bin/activate       # Mac/Linux

# Install dependencies
pip install -r requirements.txt

# Create .env file
cp .env.example .env
# Fill in your DATABASE_URL, JWT_SECRET, GROQ_API_KEY

# Seed exam questions into database
python seed_exam.py

# Start backend
uvicorn app.main:app --reload
```

### 3. Frontend setup
```bash
cd frontend
npm install

# Create .env file
cp .env.example .env
# Set VITE_API_URL=http://localhost:8000/api

npm run dev
```

### 4. Open in browser
```
Frontend  →  http://localhost:5173
API Docs  →  http://localhost:8000/docs
```

## 📡 API Endpoints

```
POST /api/auth/register          Register new user
POST /api/auth/login             Login with email/password
POST /api/auth/refresh           Refresh access token
GET  /api/auth/me                Get current user

POST /api/exam/start             Start new exam session
GET  /api/exam/questions/{sec}   Get questions for section
POST /api/exam/submit-section    Submit section answers
POST /api/exam/complete/{id}     Complete exam + calculate scores
GET  /api/exam/result/{id}       Get exam result
GET  /api/exam/history           Get user's exam history
GET  /api/exam/leaderboard       Get global leaderboard

POST /api/ai/exam-feedback       Generate AI feedback (Groq)
POST /api/code/run               Execute code (Python/Java/C/C++)
```

Full interactive docs at `http://localhost:8000/docs`

## 💡 Key Technical Decisions

**Why a local code execution engine over Piston API?**
Piston's public API became whitelist-only in Feb 2026. Building a local subprocess-based executor gives full control, zero rate limits, and works offline — better for a portfolio demo.

**Why sessionStorage for exam scores?**
The exam uses Railway for auth/persistence, but questions come from a local question bank. Storing scores locally ensures the result page always shows correct data regardless of backend response, making the demo bulletproof.

**Why Groq over OpenAI?**
Groq's free tier provides significantly faster inference with LLaMA3. For a portfolio project, it offers the best cost-to-performance ratio without needing a paid API plan.

**Why randomized question selection?**
Using `pickRandom(bank, 5)` with a 40-question bank (8x the shown questions) ensures students get different questions every attempt — making repeated practice genuinely useful.
